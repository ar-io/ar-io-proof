# Changelog — `ar-io-proof` (Python kernel)

All notable changes to the Python verification kernel. Format loosely follows
[Keep a Changelog](https://keepachangelog.com/); this project is pre-1.0, so
minor versions may include behavior changes.

## [0.3.0] — 2026-07-15

Attested-evidence-export verification — the `ario.evidence/v1` verifier, from
scratch, reaching byte-identical verdicts with the TS kernel (Evidence-Labs
export lane, evidence-export.md).

### Added

- **`ario.evidence/v1` verifier (`ario_proof.evidence`).** New
  `verify_evidence_bundle()` verifies the Ed25519 evidence wrapper and dispatches
  two bodies: `ario.anchor.trace/v1` (per-event envelope + RFC 9162 inclusion +
  checkpoint merkle-root recompute + `content_ok` tri-state, with an optional
  per-gateway on-chain re-fetch, §4.2) and `ario.evidence.export/v1` — the full
  §5 offline algorithm: source-bundle linkage, source-verdict recompute,
  cached-vs-recomputed **verdict agreement over the deterministic dimensions only**
  (on-chain excluded), embedded RSA-PSS attestation verify + operator-address +
  `data_hash = SHA-256(JCS(checkpoint.envelope))` binding + `subject_ref`
  tri-state, and the recomputed §4 verdict object. Exit-code mapping 0/1/2/3.
- **RSA-PSS-SHA-256 primitive (`ario_proof.rsa_pss`).** `verify_rsa_pss_sha256()`
  (salt length pinned to **32** = `RSA_PSS_SALTLEN_DIGEST`, MGF1-SHA-256) and
  `derive_operator_address(n) = base64url(SHA-256(modulus))`. Malformed key /
  signature raise `MalformedRsaError` (→ exit 2); a well-formed non-verifying
  signature returns `False` (→ exit 1).

### Dependencies

- **New: `cryptography>=41.0.0`** — the RSA-PSS verify primitive. (Ed25519 stays
  on PyNaCl; SHA-256/JCS unchanged.)

### Conformance

- Corpus advanced to `test-vectors-v1.3`: the attested-evidence-export set under
  `test-vectors/evidence-export/` (one positive + thirteen tampered/malformed/
  undetermined classes + a bare RSA-PSS primitive vector), generated by
  `tools/gen-vectors/gen_export_vectors.py`.
- **Cross-kernel byte-agreement:** `cross-kernel/run_export.sh` drives BOTH the
  Python and TS kernels over the shared golden export + tampered classes and
  asserts identical verdicts — including a **byte-identical recomputed §4 verdict
  object** (`SHA-256(JCS(verdict))` matches:
  `738663f32ccec765743ed7d53f79b09254af2f11656e7d9955e521c4edeab8cb`).

## [0.2.0] — 2026-06-15

Full-family verifier + `ario.events/v1` ratified (kernel-ratification lane).

### Added

- **`ario.events/v1` accepted.** `ACCEPTED_SPEC_VERSIONS` now includes
  `ario.events/v1` (external commitment + Minimal disclosure), alongside
  `ario.agent/v1` and `ario.mlflow/v1` — ratified at envelope-spec v1.3.

### Changed

- **`payload_hash` is required (envelope-spec §2).** A missing `payload_hash`
  is now a hard reject in every mode, independent of whether there is material
  to compare it against. The prior compare-only-if-present lenience is fixed
  (BDFL ruling; §2 is binding). A *present* `payload_hash` with no inline
  payload and no supplied bytes remains **undetermined** (`payload_hash_ok is
  None`) — "signature-valid, semantics-undetermined."

### Conformance

- The `ario.events/v1` corpus vectors now gate through `verify_envelope`
  (full bind with the committed record bytes; signature-only → undetermined),
  not just at the primitive level.
- Corpus advanced to `test-vectors-v1.2`: chained two-checkpoint vector +
  `negatives/` (malformed `spec_version` minors, lone UTF-16 surrogate,
  missing `payload_hash`).
